# Tauri 自动签名与 GitHub Actions 发布手册

> 适用项目：`pt-manager-tauri`

Tauri updater 需要用 minisign 私钥在 CI 中给安装包签名，并把对应公钥写入
`src-tauri/tauri.conf.json`。私钥只放在 GitHub Actions Secret 中，不能提交到仓库。

## 1. 生成签名密钥

在项目根目录执行：

```powershell
npx @tauri-apps/cli signer generate -w tauri-signing.key
```

如果不想设置密码，提示输入密码时直接按两次回车。

生成后会得到：

| 文件 | 用途 |
| --- | --- |
| `tauri-signing.key` | 私钥，用于 CI 签名，不能提交 |
| `tauri-signing.key.pub` | 公钥，用于 `tauri.conf.json` |

务必确认 `.gitignore` 已忽略 `tauri-signing.key`。

## 2. 配置 GitHub Secrets

进入 GitHub 仓库：Settings -> Secrets and variables -> Actions -> New repository secret。

| Secret 名称 | 内容 |
| --- | --- |
| `TAURI_SIGNING_PRIVATE_KEY` | `tauri-signing.key` 的完整两行内容 |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | 生成私钥时设置的密码；没设置就留空 |

`TAURI_SIGNING_PRIVATE_KEY` 必须长这样，第一行和第二行都要保留：

```text
untrusted comment: minisign secret key: XXXXXXXXXXXXXXXX
RWRTxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

不要把 `tauri-signing.key.pub`、`tauri.conf.json` 里的 `pubkey`，或自己额外 base64
过的内容填进 `TAURI_SIGNING_PRIVATE_KEY`。

## 3. 更新 tauri.conf.json 公钥

`plugins.updater.pubkey` 需要填写公钥文件内容的 base64 值：

```powershell
[Convert]::ToBase64String(
  [System.Text.Encoding]::UTF8.GetBytes(
    (Get-Content tauri-signing.key.pub -Raw)
  )
)
```

然后把输出填入：

```json
"plugins": {
  "updater": {
    "endpoints": [
      "https://github.com/Edwinzzzs2/pt-manager-tauri/releases/latest/download/latest.json"
    ],
    "pubkey": "这里填上一步输出的 base64 公钥"
  }
}
```

## 4. 常见错误

| 错误 | 原因 | 处理 |
| --- | --- | --- |
| `Missing comment in secret key` | Secret 少了第一行 `untrusted comment` | 重新粘贴完整两行私钥 |
| `Invalid padding` | 私钥第二行不是合法 base64，常见于少了末尾 `=`、多了空格、粘错文件 | 重新从 `tauri-signing.key` 复制完整两行 |
| `incorrect updater private key password` | 密码不匹配 | 检查 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` |
| 客户端更新验签失败 | `tauri.conf.json` 公钥和 CI 私钥不是一对 | 重新生成密钥对，并同步更新 Secret 与公钥 |

## 5. 本地验证

```powershell
$env:TAURI_SIGNING_PRIVATE_KEY = (Get-Content tauri-signing.key -Raw)
$env:TAURI_SIGNING_PRIVATE_KEY_PASSWORD = ""
npm run tauri -- build
```

构建成功后，`src-tauri/target/release/bundle/nsis` 目录里应有 `.exe` 和 `.sig` 文件。
